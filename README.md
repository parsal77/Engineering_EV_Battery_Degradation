## Motivation
EV battery degradation modeling is central to practical range estimation, warranty analytics, and maintenance planning. This repository demonstrates an end-to-end ML workflow from raw experimental `.mat` files to deployable model artifacts and benchmarked results.

## Dataset
This project uses the NASA Ames Prognostics Center of Excellence (PCoE) Li-ion Battery Aging Dataset.

- Batteries used: `B0005`, `B0006`, `B0007`, `B0018`
- Raw source: https://data.nasa.gov/dataset/li-ion-battery-aging-datasets
- Mirror: https://www.kaggle.com/datasets/patrickfleith/nasa-battery-dataset

Place raw files in `data/raw/` as:
- `B0005.mat`
- `B0006.mat`
- `B0007.mat`
- `B0018.mat`

Or run preprocessing notebooks/scripts; a public mirror downloader is integrated in `src/data_loader.py`.

## Key Findings
- Capacity fade is monotonic but battery-specific, with visibly different degradation slopes across `B0005/B0006/B0007/B0018`.
- Cross-battery RUL generalization is strongest for tree-based models (`RF`, `XGBoost`) in this dataset split.
- Neural sequence models are sensitive to sequence design and sample count; Transformer outperformed LSTM for RUL in this setup.
- SOH is highly predictable from engineered cycle-level features under the NASA testbed conditions.
- Thermal and voltage-derived features contribute strongly to prognostics signal quality.
